# Security Policy

## Reporting A Concern

Use the repository issue tracker for non-sensitive documentation problems.

Do not post confidential information, private city data, security-sensitive details, or unreleased vendor information in public issues.

For sensitive concerns, contact the maintainers through the project owner or the private reporting channel named by the publishing organization.

## Scope

This repository contains public toolkit documentation, prompt packs, templates, examples, evaluation guides, and review materials.

Security-sensitive review should focus on:

- accidental disclosure of private source material
- publication of local city facts that were not meant for reuse
- prompt or template language that encourages bypassing legal, records, accessibility, procurement, labor, privacy, or security review
- examples that reveal sensitive implementation details

## Publication Boundary

Keep internal notes, source-audit workpapers, private city records, vendor-sensitive information, and unresolved extraction drafts out of the public repository.
