# Code of Conduct

## Our Commitment

This project is intended to be a practical, collaborative toolkit for public-sector AI policy work.

We expect participants to communicate professionally, treat others with respect, and keep feedback focused on the work.

## Expected Behavior

- be respectful and direct
- separate critique of the artifact from critique of the person
- keep review comments actionable
- protect confidential or workspace-only material from public release
- avoid harassment, discrimination, or intimidation

## Unacceptable Behavior

- abusive or demeaning language
- personal attacks
- deliberate misrepresentation of another contributor’s work
- discriminatory comments or exclusionary conduct
- repeated disruption of review, planning, or collaboration
- publishing workspace-only internal material as if it were public-ready

## Collaboration Norms

- keep assumptions visible
- note unresolved questions clearly
- respect module boundaries and file ownership during review
- use review comments to improve the artifact, not to escalate conflict

## Reporting Concerns

If you experience or observe behavior that violates this code of conduct, report it to the maintainers through the project’s designated contact path.

Use this placeholder until the final public repo defines the actual contact channel:

- `[[MAINTAINER_CONTACT_CHANNEL]]`

## Enforcement

Project maintainers may remove comments, pause contributions, or restrict participation when behavior undermines a safe and productive collaboration space.

Enforcement should be proportionate, documented, and focused on restoring a workable review environment.

## Scope

This code of conduct applies to all public repository spaces, including:

- issues
- pull requests
- comments
- reviews
- project discussions

It does not replace any legal, employment, or organizational policy that may apply in a contributor’s own environment.
