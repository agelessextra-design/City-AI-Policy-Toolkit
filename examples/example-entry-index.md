# Example Entry Index

## Purpose

This index gives users a direct entry point into the worked-example layer.

Use it when you need to see how the toolkit works in practice rather than only reading templates, prompts, and evaluation assets in isolation.

## Current Example Types

### Governance-Family Worked Examples

Use these when you want to study the original validated governance pilot paths:

- governance source-to-outline path
- ownership/register path
- internal tools and shadow AI path

### Five-Layer System Example

Use `guide-worked-example-midsize-city-five-layer-build.md` when you want to see how a midsize city could move through all five module families from strategy to public trust.

Use `guide-worked-example-small-city-minimum-viable-five-layer-path.md` when you want to see how a small city can use the toolkit with lighter staffing and governance capacity.

Use `guide-worked-example-large-city-enterprise-five-layer-path.md` when you want to see how a large city can sequence the same five-layer system with stronger portfolio, operating-model, and public-accountability demands.

### City-Scale Starter Comparison

Use `matrix-city-scale-adaptation-starter-comparison.md` when you want to compare how small-city, midsize-city, and large-city starting conditions change the path through the toolkit.

## Annotation Rule

Every new full-system example should explicitly name the supporting:

- template artifacts in `toolkit/templates/`
- prompt packs in `toolkit/prompts/`
- evaluation assets in `toolkit/evaluations/`
- review assets in `toolkit/review-sprint-kit/`
