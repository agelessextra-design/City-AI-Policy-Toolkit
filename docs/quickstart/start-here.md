# Start Here

## Who This Toolkit Is For

This toolkit is for city teams building a municipal AI system that needs more than one policy document.

Use it when you need to coordinate:

- leadership direction
- governance and approval controls
- workforce readiness
- operational delivery
- community trust

## Pick A Starting Module

Choose the module family that matches the city’s current bottleneck.

### Start With Strategic Leadership If

- executive sponsorship is unclear
- the city needs a portfolio charter
- the city has many possible AI ideas but no sequencing logic

### Start With Governance And Policy If

- the city already wants to move but lacks approval, risk, and accountability rules
- departments are experimenting without a common control model

### Start With Workforce And Learning If

- staff adoption is already happening without a readiness plan
- managers, HR, or labor partners need a defined support model

### Start With Operations And Service Delivery If

- approved or likely use cases exist but there is no shared delivery playbook
- support, rollout, testing, or pause authority is unclear

### Start With Community Trust If

- resident-facing uses are emerging
- the city needs a public explanation, notice, accessibility, or redress approach

## Minimum Safe Workflow

1. Pick the starting module.
2. Gather local context and unresolved decisions.
3. Build the outline before drafting prose.
4. Keep placeholders and local-decision markers visible.
5. Run evaluation before human review.
6. Use examples only as examples, not as default city policy text.
