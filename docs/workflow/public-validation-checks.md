# Public Validation Checks

## Purpose

This guide defines the lightweight validation checks that should pass before a public release is cut.

## Required Checks

- required public files are present
- public links resolve
- public docs do not point to internal numbered-workspace paths
- placeholders remain visible where local decisions are required
- examples are clearly labeled and separated from reusable toolkit assets
- quickstart, workflow, evaluation, and review guidance follow the same phase order
