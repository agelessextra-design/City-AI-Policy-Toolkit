# Release Readiness

## Purpose

This guide explains the minimum conditions for treating the public toolkit bundle as release-ready.

## Release Levels

- internal alpha: navigation works and core layers are present, but some critical sections may still be below threshold
- public beta: all sections clear `85%` and the public user path is coherent
- `1.0.0`: all critical sections clear `90%` and cross-system evidence exists

## Release Blockers

- unresolved critical issue count above `0`
- public docs that point back to internal-only workspace files
- examples that are not clearly labeled as examples
- contradictions between quickstart, workflow, evaluation, and review guidance

## Evidence To Review

- public README and docs tree
- latest evaluation summary
- latest cross-system audit
- current release-gate status
