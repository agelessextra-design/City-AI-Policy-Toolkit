# Workflow

This folder should explain how the toolkit phases connect for a standalone user.

The public workflow guidance should cover:

- readiness and context discovery
- source audit and neutralization when local source-backed material is involved
- template selection by module family
- outline-first drafting
- evaluation gates
- review sprint preparation
- packaging and maintenance

It should also explain how the five module families interact:

- leadership sets direction
- governance sets controls
- workforce enables staff adoption
- operations handles delivery and monitoring
- community trust handles public explanation and feedback

It should also include:

- release-readiness guidance
- lightweight public validation checks

Do not expose internal orchestration or scratch coordination artifacts here.
