# Board Review Packet Template

## Purpose

Use this template to assemble a board-style milestone packet after a batch is stable enough for external-lens review.

This is not a city policy packet. It is a toolkit-readiness packet for simulated board-style scrutiny.

## Packet Header

- milestone batch:
- review date:
- board lens:
- packet owner:

## What The Board Is Reviewing

State:

- which sections or module families are in scope
- what the milestone claims to have achieved
- what is explicitly out of scope

## Core Evidence

Include:

1. release-gate snapshot
2. latest section score summary
3. latest cross-system audit summary
4. public-facing entry points relevant to the batch
5. open major issues that remain

If the batch involves publicly controversial, rights-adjacent, or politically sensitive AI use, also include:

6. what the city is and is not doing
7. current complaint and redress routing
8. blocked actions and unresolved decisions
9. next public reporting point

## Likely Objections

List the top objections the board reviewer is likely to raise before the session starts.

## Questions The Packet Must Answer

- can a normal user start safely?
- is the workflow coherent?
- are unresolved issues visible and routed?
- is the batch strong enough to move closer to release?

## Board Outcome Capture

Record:

- pass reasons
- fail reasons
- required follow-up work
- owner of each follow-up item
