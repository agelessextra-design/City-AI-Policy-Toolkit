# Contradiction Sweep Checklist

## Purpose

Use this checklist to confirm that a draft, outline, or review packet does not contain cross-document contradictions before it advances.

Its job is to make contradiction checking repeatable, visible, and routable.

## Sweep Goal

The sweep should answer three questions quickly:

1. Are the sections internally consistent?
2. Does the packet preserve the same meaning across artifacts?
3. If there is a contradiction, where should it go next?

## Required Checks

### Terminology

- role names match across sections
- artifact names match across companion docs
- the same concept is not renamed without a visible reason

### Numbers And Thresholds

- timing commitments match across sections
- tier definitions match across sections
- threshold language matches across sections
- score terms mean the same thing wherever they appear

### Narrative And Values

- values language is consistent
- sequencing language is consistent
- training philosophy is consistent
- trust commitments align with governance and operations claims

### Ownership And Escalation

- local decisions remain visible
- hidden source-city assumptions are visible or removed
- owner names stay consistent across the issue log, routing register, and review packet

### Review Packet Fit

- contradiction sweep result is summarized in the packet
- failing criteria are tied to issue-log entries
- decision-worthy contradictions are routed to the next owner

## Severity Guidance

Classify each finding before routing it.

- `critical` if it changes meaning or blocks readiness
- `major` if it should be fixed before sprint review
- `moderate` if it should be fixed during the current revision loop
- `minor` if it is cosmetic but should still be recorded

## Output Fields

- artifact reviewed
- review date
- reviewer
- contradiction sweep status
- failing criteria
- rework notes
- next action
- owner
- final verdict

## Outcomes

- `clean`
- `clean with flags`
- `blocked by contradiction`
- `blocked pending local decision`

## Working Rules

- do not normalize contradictory text by selecting the nearest wording
- do not clear a contradiction without a visible owner or decision record
- do not let a score override a failed contradiction sweep
- do not treat the checklist as a rewrite tool
