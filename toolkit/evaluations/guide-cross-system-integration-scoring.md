# Cross-System Integration Scoring Guide

## Purpose

This guide defines how to score integration across sections after local section quality improves.

Use it to evaluate whether the toolkit behaves like one coherent system rather than a pile of individually strong files.

## Integration Checks

Score these checks on a `0` to `4` scale.

| Check | What To Look For |
|---|---|
| Terminology alignment | role names, outcome states, and artifact names stay consistent |
| Workflow order | intake, outline, draft, evaluation, review, and packaging sequence remains stable |
| Placeholder discipline | placeholders and local-decision markers mean the same thing across layers |
| Handoff quality | artifacts pass enough information to the next stage without re-deriving context |
| Family coverage | all five module families can traverse the shared workflow |
| Public path coherence | README, onboarding, templates, prompts, evaluations, and examples point to each other cleanly |

## Integration Outcome Bands

- `90 to 100`: cross-system accepted
- `85 to 89`: nearly ready
- `80 to 84`: usable but still risky
- `< 80`: integration loop required

## Required Output

Return:

- check-by-check scores
- overall integration percent
- contradiction summary
- sections affected
- next repair owner
