# Evaluation Entry Index

## Purpose

This index gives users a direct entry point into the evaluation layer.

Use it when you need to choose the right evaluation artifact without re-reading the entire evaluation framework.

## Core Shared Assets

- `template-shared-issue-log.md`
- `template-module-family-scorecard.md`
- `template-review-readiness-summary.md`
- `guide-module-family-review-readiness.md`
- `template-board-review-scorecard.md`
- `guide-cross-system-integration-scoring.md`

## When To Use Which Asset

### Evaluate A Single Artifact Or Module Family

Use:

- `guide-module-family-review-readiness.md`
- `template-module-family-scorecard.md`
- `template-review-readiness-summary.md`
- `template-shared-issue-log.md`

### Evaluate Cross-System Integration

Use:

- `guide-cross-system-integration-scoring.md`
- `template-shared-issue-log.md`

### Simulate Board Review

Use:

- `template-board-review-scorecard.md`
- `template-shared-issue-log.md`

### Review A Governance Pilot Artifact

Use the shared assets first, then add any local governance-specific rubrics or issue logs your city maintains.

## Selection Rule

- use shared assets unless a family-specific checklist or rubric is clearly required
- do not create a new scoring schema when the shared schema is sufficient
